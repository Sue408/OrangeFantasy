# OrangeFantasy项目开发文档

## 1 技术栈

### 1.1 后端

- **框架**：FastAPI (python ~= 3.10)
- **数据库**：PostgreSQL (生产环境) / SQLite (开发环境)
- **AI集成**：OpenAI SDK (MVP阶段手动组装Prompt)
- **类型验证**：Pydantic
- **流式输出**：WebSocket
- **用户认证**：pyjwt + OAuth2
- **密码哈希**: bcrypt
- **ORM支持**：SQLAlchemy
- **开发环境管理**：uv

### 1.2 前端


- **框架**：vue3 (node.js ~= 24)
- **SPA支持**：vue-router
- **全局状态管理**：pinia
- **HTTP网络请求**：Axios
- **WebSocket支持**：原生WebSocket
- **开发环境管理**: npm

### 后端设计

#### 依赖链

```
加载核心配置类 (读取.env配置内容)
  ↓
加载数据库初始化模块 (连接数据库并导出Engine、SessionFactory、Base对象)
  ↓
加载数据库模型定义模块 (导出ORM模型)
  ↓
加载Pydantic模型定义模块 (导出Pydantic模型) & 加载工具模块 & 加载AI业务模块
  ↓
加载API定义模块
  ↓
main.py启动
```

#### 核心配置类实现

```python
from pydantic_settings import BaseSettings

class DatabaseSetting(BaseSettings):
	""""数据库配置"""
	host: str = '.test.db'
	name: str = 'sqlite'
	
class SecuritySetting(BaseSettings):
	"""安全验证配置"""
	key: str = "11111111111111111111111111111111"

class Setting(BaseSettings):
	"""核心配置类定义"""
	# ========= 后端服务配置 =========
	debug: bool = True
	host: str = "localhost"
	port: int = 8080
	
	# ========= 数据库配置 =========
	database: DatabaseSettings = DatabaseSetting()
	
	# ========= 安全验证配置 =========
	security: SecuritySetting = SecuritySetting()
	
	class Config:
	"""读取配置"""
	env_file = [".env.local", ".env"]
	env_file_encoding = 'utf-8'
	
config: Setting = Setting() # 实例化配置对象
```

### 前端设计

#### 路由结构与规则

##### 路由结构

```
home
├ introduce # 介绍页面
├ creation # “我的作品”页面
└ working # 创作页面
auth # 登录页面路由
```

##### 路由规则

1. 未登录状态只能访问introduce、auth两个页面

#### 文件组织规则

1. 路由指向pages文件夹下的vue页面文件，并且页面文件的组织规则需要符合路由结构，如"Working.vue"属于home的子路由，因此"working.vue"需要放置在home文件夹下。
2. 除common文件夹外，其余组件按照所属的页面文件进行分文件夹进行放置，如"TextEdit.vue"被用于"Working.vue"，因此"TextEdit.vue"需要放置在working文件夹下。
3. 需要确保文件夹名称采用小写，单词之间采用"-"连接，vue文件采用"帕斯卡命名法"，其余文件默认采用" 驼峰式命名法"。
4. services文件夹用于对接后端的API并进行封装，必须确保对应关系，如"authServices.ts"定义的API调用方法需严格与后端"auth.py"中定义的API接口对齐。

## 数据库结构设计

### Users

#### 说明

用户表，储存用户关键信息

#### 字段

- id<type: Integer>: 主键
- nickname<type: String>: 昵称/笔名，非空
- avator<type: Text>: 头像 (URL/Base64)
- phone<type: String>: 手机号码，非空，唯一
- password_hash<type: String>: 密码哈希，非空

#### 关系

- novels: 绑定Novels表，一对多关系

### Novels

#### 说明

作品表，储存作品关键信息

#### 字段

- id<type: Integer>: 主键
- user_id<type: Integer>: 外键，关联User主键，非空
- name<type: String>: 作品名称，非空
- type<type: String>: 类型，非空 (枚举: "short"/"long")

#### 关系

- user: 绑定Users表，多对一关系
- capters: 绑定Capters表，一对多关系

### Capters

#### 说明

章节表，储存作品内容信息

#### 字段

- id<type: Integer>: 主键
- novel_id<type: Integer>: 外键，关联Novel主键，非空
- name<type: String>: 章节名称，非空
- number<type: Integer>: 章节序号
- step<type: String>: 所属卷名称，默认为空值
- content<type: Text>: 文本内容 (实际考虑下单章节的文本内容应小于3万字)，非空

#### 关系

- novel: 绑定Novels表，多对一关系
